# Roadmap

Open Source Starter Lab is moving from a beginner practice repo into a contributor platform.

The roadmap is intentionally practical: every item should make it easier for beginners to start, for maintainers to trust the work, or for contributors to show proof.

## Now

- Keep good first issues small, clear, and reviewable.
- Maintain the live website and README visual preview.
- Keep Automation Health, CI, Daily Issue Bot, Contributor Queue, and PR Welcome Guard green.
- Create or update Open Source Trust Passports after real merged PRs, including Levels 1, 2, and 3.
- Use the weekly assignment thread and First PR Cohort to match contributors to issues.
- Keep the generated maintainer dashboard and monthly contributor spotlight refreshed.

## Next

- Add more examples of high-quality PR descriptions.
- Improve issue templates with clearer proof fields.
- Turn the website into the main product surface for issue discovery, contributor proof, and passport browsing.

## Later

- Generate visual passport cards from passport files.
- Add richer maintainer dashboard signals for waiting contributors, open PRs, and stale assignments.
- Add optional JSON exports for public contributor proof.
- Add a small release process for major repo milestones.

## Not Planned

- Fake stars, fake forks, or fake activity.
- Spam comments in unrelated repositories.
- Large beginner tasks that are hard to review.
- Automated merging without human review.

## Decision Rule

If a feature does not help contributors start, finish, prove, or continue, it should wait.
