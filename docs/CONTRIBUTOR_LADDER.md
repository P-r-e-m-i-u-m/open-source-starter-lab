# Contributor Ladder

This repo is designed so contributors can grow from first PR to maintainer-style work.

For the command-line version, see [Contributor Progress Path](CONTRIBUTOR_PROGRESS_PATH.md).

## Level 1: First PR

Good tasks:

- Fix unclear wording.
- Add a missing link.
- Add a contributor card.
- Improve one command example.

Proof to include:

- `npm run check` output.

## Level 2: Documentation Builder

Good tasks:

- Add a short Git/GitHub guide.
- Add examples for common errors.
- Improve a checklist.
- Make a guide easier to scan.

Proof to include:

- Mention which guide changed.
- Confirm links and commands are correct.

## Level 3: CLI Contributor

Good tasks:

- Add a CLI smoke test.
- Improve command output.
- Add validation for bad input.
- Add one small command or flag.

Proof to include:

- `npm run check`
- The exact CLI command you tested.

## Level 4: Community Helper

Good tasks:

- Answer a Discussion clearly.
- Improve an issue description.
- Help a contributor understand a failed check.
- Suggest a smaller scope for a broad PR.

Proof to include:

- Link the issue, PR, or Discussion.

## Level 5: Maintainer Practice

Good tasks:

- Review a small PR.
- Improve an issue template.
- Add a maintainer reply example.
- Improve the daily issue backlog.

Proof to include:

- Explain the review or maintainer problem you solved.

## Review Style

Good reviews are specific, kind, and actionable. Point to the exact problem, explain the reason, and suggest the next step.
