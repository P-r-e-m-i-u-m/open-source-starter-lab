# Contribution Glossary

New to open source? These words show up often in GitHub issues, pull requests, and reviews.

## fork

A fork is your own copy of someone else's repository on GitHub. You can change your fork without changing the original project.

## branch

A branch is a separate line of work in a repository. Use a branch when you want to make one focused change without touching the main version directly.

## commit

A commit is a saved checkpoint of your changes. It records what changed and includes a short message explaining why.

## pull request

A pull request, often called a PR, is a request to merge your changes into another branch or repository. It gives maintainers a place to review, discuss, and approve your work.

## issue

An issue is a GitHub thread for a task, bug, question, or idea. Contributors use issues to understand what needs to be done before opening a pull request.

## review

A review is feedback on a pull request. A maintainer or contributor may approve the change, ask questions, or request updates before it is merged.

## CI

CI means continuous integration. It usually runs checks like builds or tests automatically so everyone can see whether a pull request still works.
