I added a Daily Issue Bot to Open Source Starter Lab.

The idea is simple:
beginner-friendly open source repos should not only say “contributions welcome.”
They should give people clear, small, finishable work.

So I built a bot that creates one starter issue per day from a curated backlog.

Each issue includes:
- context
- a clear goal
- suggested files
- acceptance criteria
- beginner-friendly labels

It is not trying to fake activity.
It is a maintainer helper that keeps the repo ready for new contributors.

If you are learning GitHub, open source, or your first PR workflow, this repo is for practice.

Repo:
https://github.com/P-r-e-m-i-u-m/open-source-starter-lab
