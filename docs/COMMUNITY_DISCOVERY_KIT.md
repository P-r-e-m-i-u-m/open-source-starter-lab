# Community Discovery Kit

Use this page when sharing Open Source Starter Lab in places where beginners ask how to make their first pull request.

The goal is to help people, not spam. Share only where beginner resources are welcome, and be clear that the repo is a practice lab.

## One-Line Pitch

Open Source Starter Lab is a beginner GitHub project where people can practice their first pull request, first open-source contribution, good first issue, CI checks, and maintainer review in one small repo.

## Short Post

```md
I built Open Source Starter Lab for people who want to make their first open-source pull request but do not know where to start.

It has:

- good first issues
- beginner-friendly tasks
- a Start Here guide
- CI checks
- a First Merge Wall
- a Discussion where you can introduce yourself and get an issue suggestion
- skill-based first issues for HTML/CSS, JavaScript, Python, docs, testing, and Git

Repo:
https://github.com/P-r-e-m-i-u-m/open-source-starter-lab
```

## DEV / Hashnode Post Idea

Title:

```text
I built a small GitHub repo for your first open-source pull request
```

Outline:

1. The first contribution is confusing because setup, issue choice, Git, CI, and maintainer communication all happen at once.
2. Good first issues are useful only when they are current, scoped, and reviewed.
3. Open Source Starter Lab keeps the codebase small and the workflow real.
4. New contributors can introduce themselves and get matched to a task.
5. After merge, contributors can add themselves to the First Merge Wall.

## Where To Share

Good places:

- Reddit threads where someone asks how to start open source
- DEV Community posts about first pull requests
- Hashnode posts about GitHub beginner workflows
- Discord/Slack channels that allow beginner resources
- GitHub Discussions about first contributions
- Hacktoberfest preparation threads when the season opens

Avoid:

- Posting unrelated comments in big repos
- Asking for stars
- Posting the same message everywhere
- Claiming the repo guarantees jobs, badges, or accepted PRs

## What Makes This Repo Different

Open Source Starter Lab is not just a list of names. It gives beginners a real workflow:

1. Choose a small issue.
2. Comment before starting.
3. Make a focused change.
4. Run checks.
5. Open a pull request with proof.
6. Get review feedback.
7. Add the first merge to the wall.
