# Second PR Path

Your first merged pull request proves you can finish the GitHub workflow.

The second pull request is different. It should show that you can make a small improvement with a little more ownership.

## Pick One Route

| Route | Good next task | Search |
| --- | --- | --- |
| Docs route | Improve one guide, checklist, or example | [30 minute docs issues](https://github.com/P-r-e-m-i-u-m/open-source-starter-lab/issues?q=is%3Aissue+is%3Aopen+label%3A%22time%3A+30+min%22+label%3Adocumentation+no%3Aassignee) |
| Code route | Improve one CLI message or command | [CLI issues](https://github.com/P-r-e-m-i-u-m/open-source-starter-lab/issues?q=is%3Aissue+is%3Aopen+label%3Acli+no%3Aassignee) |
| Testing route | Add one small smoke test or assertion | [Testing issues](https://github.com/P-r-e-m-i-u-m/open-source-starter-lab/issues?q=is%3Aissue+is%3Aopen+label%3Atesting+no%3Aassignee) |
| Trust route | Improve one issue so another beginner can finish it | [Community issues](https://github.com/P-r-e-m-i-u-m/open-source-starter-lab/issues?q=is%3Aissue+is%3Aopen+label%3Acommunity+no%3Aassignee) |

## What Changes After The First PR

For your first PR, it is fine to make the smallest safe change.

For your second PR, try to include:

- one clear reason the change helps the next contributor
- one proof command, usually `npm run check`
- one before/after example when the change affects wording or CLI output
- one linked issue using `Closes #issue-number` when possible

## Good Second PR Examples

- Add one missing test assertion for an existing CLI behavior.
- Improve a confusing command output message.
- Turn a vague issue into a clearer checklist.
- Add one example to a guide that already exists.
- Help one beginner in Discussions with exact commands.

## Avoid This

- Do not rewrite many files in one PR.
- Do not change automation without explaining how you tested it.
- Do not open a huge feature after only one small docs change.
- Do not use AI output blindly. Read the repo and keep the change useful.

## Comment To Claim

```md
I finished one PR and would like to take this as my second PR.

I will keep it focused and run `npm run check`.
```

## Maintainer Promise

If your first PR was merged here, you are welcome to ask for a second task. The goal is not to keep you busy. The goal is to help you become a more reliable contributor one small PR at a time.
