# Issue Quality Scoring

Open Source Starter Lab should feel curated, not random.

Every daily starter issue gets a small quality score before it is posted. The score is not for showing off. It protects beginners from vague tasks that waste time.

## Score Signals

An issue earns points when it has:

- clear context
- a focused goal
- suggested files
- at least three acceptance criteria
- a time label
- a contributor level label
- helpful notes written in a human voice

## Score Meaning

| Score | Meaning | Maintainer action |
| --- | --- | --- |
| 95-100 | Excellent | Ready to post |
| 80-94 | Strong | Usually okay, but check wording |
| Below 80 | Needs polish | Split or rewrite before assigning |

## Maintainer Rule

If a new contributor cannot answer these questions in under one minute, the issue is not ready:

- What file should I open first?
- What should I change?
- What proof should I put in the PR?
- What should I avoid changing?

## Why This Matters

Beginner repos often fail because the issue looks easy but hides missing context. This repo should do the opposite: small scope, obvious first step, visible finish line.
