# Contributor Passports

This folder stores Open Source Trust Passport files.

A passport is created or updated after a contributor gets a reviewed pull request merged.

## Current Passport Rules

- One file per contributor.
- File name uses the GitHub username in lowercase.
- Each entry should link to a real merged PR number.
- The passport should stay short, useful, and proof-based.

## Why This Exists

Many beginners need more than a merged PR. They need a simple proof artifact they can share on GitHub, LinkedIn, X, or a resume.

Maintainers also need better signals than "this person opened a PR." A passport records whether someone shipped a focused change, followed checks, and completed the review path.
