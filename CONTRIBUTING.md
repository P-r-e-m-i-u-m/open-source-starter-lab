# Contributing

Thanks for helping improve Open Source Starter Lab.

## First-Time Contributor Path

1. Find an issue labeled `good first issue`.
2. Comment that you want to work on it.
3. Create a branch.
4. Make one focused change.
5. Run `npm run check`.
6. Open a pull request and include your test result.

## Commands

```bash
npm install
npm run build
npm test
npm run demo
npm run check
```

## Pull Request Checklist

- [ ] My change is small and focused.
- [ ] I ran `npm run check`.
- [ ] I explained what changed.
- [ ] I linked the issue if one exists.
- [ ] I used clear, respectful wording.

## Good Contribution Ideas

- Add beginner docs.
- Improve examples.
- Fix unclear wording.
- Add common Git errors.
- Add test coverage for CLI behavior.
- Improve issue templates.

## Contributor Cards

You can add a short card in `contributors/your-github-username.md`:

```md
# @your-github-username

- Learning: Git, GitHub, open source
- First contribution goal: improve one docs page
- Favorite command so far: git status
```

Keep it friendly and professional.
